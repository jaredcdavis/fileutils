Fileutils Library

Fileutils is a library of file-system related functions for Common Lisp. It is meant to be portable across Unix-like and Windows file systems and across Lisp implementations.

Fileutils avoids the Common Lisp pathname functionality (which differs across Lisp implementations), and instead provides an API where paths are represented as ordinary strings. Users of fileutils should not need to know anything about Common Lisp's pathnames.

Definitions

Configuration

Many Fileutils can optionally accept a config argument that governs their behavior. For instance, the absolute-path-p function can be configured to act like a Unix system or a Windows system.

By default all such Fileutils functions read their configuration from fileutils:*default-config*, which is automatically initialized when fileutils is loaded as is appropriate for the host.

BOZO describe configuration objects and what can be controlled.

Vanilla Path

Many Fileutils functions accept vanilla path arguments. A vanilla path is an ordinary Common Lisp strings that should generally indicate a single file or directory.

Characters like * and ?, which are given special interpretation in many shells, have no special meaning in vanilla paths. That is, a vanilla path such as "foo/*.txt" does not refer to all of the files in foo/ that end with .txt; instead, it refers to a single file whose name might have to be written using quotes like 'foo/*.txt' to avoid wildcard expansion in most shells.

The ~ character, which expands to user home directories on most shells, also has no special meaning in a vanilla path. That is, a vanilla path such as "~jared/foo.txt" does not refer to foo.txt in my home directory, but instead referse to a single file whose name might be written at the terminal as './~jared/foo.txt'.

Path Name Functions

(absolute-path-p path [:config config]) → bool
  • path is a vanilla path
  • config is a configuration which defaults to fileutils:*default-config*

We check whether path is an absolute path, returning the answer as t or nil. On Unix-like systems this essentially just means the path starts with a / character. On Windows it means the path begins with a drive letter like c:\ or a share name like \\mediabox\movies. See also relative-path-p.

(relative-path-p path [:config config]) → bool
  • path is a vanilla path
  • config is a configuration which defaults to fileutils:*default-config*

This is just the opposite of absolute-path-p.
